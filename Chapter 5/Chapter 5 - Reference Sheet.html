Penetration Testing Process

  1. Reconnaissance: Information gathering
  2. Initial Access: Exploit vulnerabilities
  3. Privilege Escalation: Increase access
  4. Lateral Movement: Expand to other systems
  5. Persistence: Maintain access
  6. Cleanup: Remove tools/traces

Reconnaissance Types

Passive Reconnaissance

No direct target engagement
  • WHOIS lookups
  • DNS queries
  • Web searches
  • Public website reviews
  • OSINT (Open Source Intelligence)

Active Reconnaissance

Direct target engagement
  • Port scanning
  • Footprinting
  • Vulnerability scanning
  • War driving/War flying

    Audits and Assessments

    Security Tests

    • Purpose: Verify control functioning
    • Performer: Internal teams
    • Frequency: Regular/automated
    • Scope: Specific controls

    Security Assessments

    • Purpose: Comprehensive review
    • Performer: Internal or external
    • Output: Risk assessment report
    • Audience: Management

    Security Audits

    • Purpose: Independent evaluation
    • Performer: Independent auditors
    • Output: Attestation
    • Audience: Third parties

    Audit Types

    Type Characteristics
    Internal Internal audit staff, independent reporting
    External Independent firm, high credibility
    Third-party Initiated by regulators/customers

    Key Acronyms & Terms

    SCAP Framework

    • CCE: Common Configuration Enumeration
    • CPE: Common Platform Enumeration
    • CVE: Common Vulnerabilities and Exposures
    • CVSS: Common Vulnerability Scoring System
    • XCCDF: Extensible Configuration Checklist Description Format
    • OVAL: Open Vulnerability and Assessment Language

    Testing Concepts

    • False Positive: Scanner reports non-existent vulnerability
    • False Negative: Scanner misses actual vulnerability
    • RoE: Rules of Engagement
    • OSINT: Open Source Intelligence
    • SOC: Service Organization Controls
    • SSAE 18: Attestation standard

    Penetration Testing

    • Lateral Movement: Spread to other systems
    • Privilege Escalation: Increase access level
    • Persistence: Maintain access
    • Footprinting: OS/service identification
    • War Driving: Wireless network reconnaissance
    • Pivoting: Using compromised system as attack platform

    Vulnerability Life Cycle

    1. Identification
    Scans, tests, disclosure
    →
    2. Analysis
    Confirm, prioritize, CVSS
    →
    3. Response
    Patch, segment, controls
    →
    4. Validation
    Rescan, audit, verify
    →
    5. Reporting
    Document, communicate

    Vulnerability Response Options

    • Patching: Apply vendor fixes
    • Segmentation: Isolate affected systems
    • Compensating Controls: Alternative protections
    • Insurance: Transfer financial risk
    • Exceptions/Exemptions: Formal risk acceptance

    Validation Methods

    • Rescanning: Verify vulnerability resolution
    • Audit: Independent verification
    • Verification: Manual confirmation
