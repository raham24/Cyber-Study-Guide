Detailed Answer Explanations

Question 1
Correct Answer: C. Threat hunting
Explanation: Threat hunting assumes that an organization has already been compromised (presumption of compromise) and actively searches for evidence of successful attacks. It uses the attacker mindset to identify what attackers might do and what evidence they might leave behind.
Why other answers are incorrect:
A. Vulnerability scanning - Identifies potential vulnerabilities, doesn't assume compromise
B. Penetration testing - Tests security controls by attempting to exploit vulnerabilities
D. War driving - Reconnaissance technique for identifying wireless networks
Key Concept: Threat hunting builds on intelligence fusion, using threat feeds, advisories, and bulletins to search for indicators of compromise.
Question 2
Correct Answer: D. Read-only
Explanation: Credentialed scans should use read-only accounts following the principle of least privilege. Since credentialed scans typically only retrieve information and don't make changes to servers, read-only access is sufficient and reduces security risk.
Why other answers are incorrect:
A. Domain administrator - Excessive privileges, violates least privilege principle
B. Local administrator - More privileges than necessary for scanning
C. Root - Highest privilege level, unnecessary and risky for scanning
Key Concept: Credentialed scanning improves accuracy by accessing system configuration information while maintaining security through least privilege.
Question 3
Correct Answer: C. Run the scan in a test environment
Explanation: Dangerous (intrusive) plug-ins may disrupt production systems or damage content. Best practice is to test these plug-ins in a test environment first to identify potential issues before running on production systems.
Why other answers are incorrect:
A. Run against production - Risk of business disruption
B. Run during business hours - Maximizes potential impact if problems occur
D. Do not run the scan - Misses potential vulnerabilities that attackers could exploit
Key Concept: Intrusive plug-ins can cause system instability, so test environments provide safe testing grounds for dangerous scans.
Question 4
Correct Answer: C. Low
Explanation: In CVSS, the Attack Complexity (AC) metric has two values: Low (L) indicates the attack does not require specialized conditions and is simpler to exploit, while High (H) indicates specialized conditions are required.
Why other answers are incorrect:
A. High - Indicates more difficult exploitation requiring specialized conditions
B. Medium - Not a valid CVSS Attack Complexity value
D. Severe - Not a valid CVSS Attack Complexity value
Key Concept: CVSS Attack Complexity only has two values: Low (easier) and High (harder).
Question 5
Correct Answer: A. False positive
Explanation: A false positive occurs when a scanner reports a vulnerability that does not actually exist. In this case, the scanner reported a vulnerability on a system that was already properly patched.
Why other answers are incorrect:
B. False negative - When scanner misses an actual vulnerability
C. True positive - Correctly identifies an existing vulnerability
D. True negative - Correctly reports no vulnerability exists
Key Concept: False positives require time-consuming administrator investigation and can be reduced through proper scanner configuration and credentialed scanning.
Question 6
Correct Answer: B. Integrity
Explanation: SQL injection allowing students to alter grades is an integrity attack. The CIA triad includes Confidentiality, Integrity, and Availability. Preventing unauthorized data modification requires integrity controls.
Why other answers are incorrect:
A. Confidentiality - Protects against unauthorized disclosure, not modification
C. Alteration - Not a standard security control category
D. Availability - Protects against service disruption, not data modification
Key Concept: Integrity controls prevent unauthorized modification of data and include input validation, access controls, and database constraints.
Question 7
Correct Answer: C. Metasploit
Explanation: Metasploit is an exploitation framework used for actively exploiting vulnerabilities, not for reconnaissance. It's used after vulnerabilities are identified to test if they can be exploited.
Why other answers are incorrect:
A. Nmap - Port scanning tool commonly used in reconnaissance
B. Nessus - Vulnerability scanner used to identify weaknesses
D. Nslookup - DNS lookup tool for gathering domain information
Key Concept: Reconnaissance focuses on information gathering, while exploitation tools like Metasploit are used after vulnerabilities are identified.
Question 8
Correct Answer: A. Patch management
Explanation: The vulnerability description specifically mentions Microsoft Advisory MS17-010 and references patches to fix the SMB vulnerabilities. This is clearly a patch management issue where proper patching would have prevented the vulnerability.
Why other answers are incorrect:
B. File integrity monitoring - Monitors file changes, doesn't prevent vulnerabilities
C. Intrusion detection - Detects attacks, doesn't prevent vulnerabilities
D. Threat hunting - Searches for existing compromises, doesn't prevent vulnerabilities
Key Concept: Patch management is a fundamental security control that addresses known vulnerabilities by applying vendor-provided fixes.
Question 9
Correct Answer: B. Web application vulnerability scanner
Explanation: Cross-site scripting (XSS) is a web-specific vulnerability that is best detected by specialized web application vulnerability scanners designed to test for web application flaws like XSS, SQL injection, and CSRF.
Why other answers are incorrect:
A. Static application test - May detect some XSS but less effective than web scanners
C. Intrusion detection system - Detects active attacks, not vulnerabilities
D. Network vulnerability scanner - Focuses on infrastructure, not web application flaws
Key Concept: Web application scanners are specialized tools that test for web-specific vulnerabilities through dynamic testing and fuzzing.
Question 10
Correct Answer: A. Lateral movement
Explanation: Lateral movement (also called pivoting) occurs when an attacker uses a compromised system to gain access to other systems on the same network. Deploying a toolkit to access additional systems is a classic example of lateral movement.
Why other answers are incorrect:
B. Privilege escalation - Increasing access levels on the same system
C. Footprinting - Reconnaissance technique for gathering system information
D. OSINT - Open source intelligence gathering from public sources
Key Concept: Lateral movement is a key phase in penetration testing and attack progression, expanding the attack scope across the network.
Question 11
Correct Answer: A. External audit
Explanation: This is an external audit because it's being performed by independent auditors to validate financial statements for external stakeholders (investors). External audits provide third-party validation for external audiences.
Why other answers are incorrect:
B. Penetration test - Tests security controls through simulated attacks
C. Internal audit - Performed by internal audit staff for internal purposes
D. Informal audit - Not a standard audit classification
Key Concept: External audits provide independent validation for external stakeholders and result in formal attestations about control effectiveness.
Question 12
Correct Answer: C. Bug bounty
Explanation: Bug bounty programs are designed to incentivize external security researchers by offering financial rewards for discovering and reporting vulnerabilities. They solicit participation from external experts through monetary incentives.
Why other answers are incorrect:
A. Threat hunting - Internal activity searching for existing compromises
B. Penetration testing - Usually performed by hired professionals, not rewarded external experts
D. Vulnerability scanning - Automated tool-based activity, not human expert participation
Key Concept: Bug bounty programs leverage the crowd-sourced security community by providing financial incentives for vulnerability discovery.
Question 13
Correct Answer: D. Persistence
Explanation: Installing a backdoor to maintain future access is establishing persistence. Persistence mechanisms ensure continued access to compromised systems even if the original vulnerability is patched.
Why other answers are incorrect:
A. Privilege escalation - Increasing access levels on the current system
B. Lateral movement - Moving to other systems in the network
C. Maneuver - General term, not specific to maintaining access
Key Concept: Persistence is a critical phase in penetration testing that simulates how real attackers maintain long-term access to compromised environments.
Question 14
Correct Answer: C. WHOIS lookups
Explanation: WHOIS lookups are passive reconnaissance because they gather information from public databases without directly engaging the target. Passive techniques don't interact with target systems.
Why other answers are incorrect:
A. Port scans - Active technique that directly probes target systems
B. Vulnerability scans - Active technique that tests target systems
D. Footprinting - Can be active or passive, but often involves direct target interaction
Key Concept: Passive reconnaissance gathers information without alerting the target, while active reconnaissance directly engages target systems.
Question 15
Correct Answer: B. CVE
Explanation: Common Vulnerabilities and Exposures (CVE) provides a standard nomenclature for describing security-related software flaws. Each vulnerability gets a unique CVE identifier for consistent reference.
Why other answers are incorrect:
A. CPE - Common Platform Enumeration for product names and versions
C. CVSS - Common Vulnerability Scoring System for rating severity
D. CCE - Common Configuration Enumeration for configuration issues
Key Concept: CVE provides standardized vulnerability naming, while CVSS provides standardized vulnerability scoring within the SCAP framework.
Question 16
Correct Answer: C. Known environment test
Explanation: When clients provide full system details in advance, this is a known environment test (formerly called "white box"). Testers have complete information about the target environment.
Why other answers are incorrect:
A. Partially known environment - Only some information provided
B. Detailed environment test - Not a standard penetration testing classification
D. Unknown environment test - No information provided (black box testing)
Key Concept: Known environment tests allow comprehensive testing but may not reflect real-world attacker perspectives, while unknown environment tests provide realistic attack simulation.
Question 17
Correct Answer: C. Rules of engagement
Explanation: Rules of engagement (RoE) define what activities are permitted during a penetration test, including whether social engineering is allowed. This document specifies the scope and boundaries of testing activities.
Why other answers are incorrect:
A. Contract - Legal agreement but typically doesn't contain detailed testing procedures
B. Statement of work - Defines deliverables but not specific testing boundaries
D. Lessons learned report - Post-test documentation, not pre-test guidance
Key Concept: Rules of engagement are critical for defining testing boundaries and ensuring legal and appropriate penetration testing activities.
Question 18
Correct Answer: B. Footprinting
Explanation: Footprinting is specifically designed to identify operating systems and applications in use on target systems. It's the most direct method for determining OS information during penetration testing.
Why other answers are incorrect:
A. Port scanning - Identifies open ports/services but doesn't directly reveal OS
C. Vulnerability scanning - May reveal OS but primarily focuses on vulnerabilities
D. Packet capture - Captures network traffic for analysis, indirect OS identification
Key Concept: Footprinting uses various techniques to identify target system characteristics including OS, applications, and network services.
Question 19
Correct Answer: B. Medium
Explanation: According to the CVSS Qualitative Severity Rating Scale, a score of 6.5 falls into the Medium category (4.0-6.9). This is a fundamental CVSS scoring concept for the Security+ exam.
Why other answers are incorrect:
A. Low - CVSS scores 0.1-3.9
C. High - CVSS scores 7.0-8.9
D. Critical - CVSS scores 9.0-10.0
Key Concept: CVSS severity scale: None (0.0), Low (0.1-3.9), Medium (4.0-6.9), High (7.0-8.9), Critical (9.0-10.0).
Question 20
Correct Answer: C. PR
Explanation: The Privileges Required (PR) metric describes the type of account access an attacker needs to execute an attack. Values include None (N), Low (L), and High (H) privilege requirements.
Why other answers are incorrect:
A. AV - Attack Vector describes how the vulnerability is exploited (Physical, Local, Adjacent, Network)
B. C - Confidentiality describes information disclosure impact
D. AC - Attack Complexity describes difficulty of exploitation
Key Concept: CVSS metrics include four exploitability measures (AV, AC, PR, UI), three impact measures (C, I, A), and one scope measure (S).

Study Recommendations

Focus Areas for Additional Study:
  • CVSS Metrics: Memorize all eight metrics and their values
  • Penetration Testing Phases: Understand reconnaissance, initial access, privilege escalation, lateral movement, persistence, and cleanup
  • False Positive vs. False Negative: Critical concept for vulnerability scanning
  • Passive vs. Active Reconnaissance: Know which techniques fall into each category
  • Audit Types: Distinguish between internal, external, and third-party audits
  • SCAP Components: Understand CVE, CVSS, CPE, CCE, XCCDF, and OVAL
