1. Cybersecurity Objectives - The CIA Triad Basic

Core Principle

Security professionals are responsible for protecting the confidentiality, integrity, and availability of information and information systems used by their organizations.

The CIA Triad Components

Confidentiality

  • Implementation Controls: Firewalls, access control lists (ACLs), encryption protocols
  • Primary Threat: Data exfiltration
  • Common Vulnerabilities: Misconfigured access controls, lost devices, insider threats

Integrity

  • Implementation Controls: Hashing algorithms, integrity monitoring solutions, checksums
  • Example Threats: Modifying financial records, bit flips, accidental changes

Availability

  • Implementation Controls: Fault tolerance, clustering, backup systems, redundancy
  • Common Threats: DDoS attacks, hardware failures, natural disasters

Nonrepudiation

  • Primary Implementation: Digital signatures
  • Purpose: Verification of origin and accountability
  • Use Cases: Email authentication, transaction verification, audit trails
Remember the CIA triad components (Confidentiality, Integrity, Availability) and understand that nonrepudiation provides assurance that actions cannot be denied. These four concepts form the foundation of information security.

2. Data Breach Risks - The DAD Triad Basic

DAD Triad

  • Disclosure: Violates Confidentiality
  • Alteration: Violates Integrity
  • Denial: Violates Availability
DAD Threat CIA Principle Violated Description Examples
Disclosure Confidentiality Exposure of sensitive information to unauthorized individuals Data exfiltration, misconfigured access controls, lost devices
Alteration Integrity Unauthorized modification of information Fraudulent transactions, website defacement, database tampering
Denial Availability Disruption of legitimate user access DDoS attacks, ransomware, outages

Five Categories of Breach Impact

Impact Categories

  • Financial Risk: Monetary damage, direct/indirect costs
  • Reputational Risk: Loss of goodwill, trust, and loyalty
  • Strategic Risk: Threatens ability to meet goals
  • Operational Risk: Disrupts day-to-day functions
  • Compliance Risk: Violates legal/regulatory requirements

Identity Theft Considerations

  • Social Security numbers
  • Bank account and credit card information
  • Driver's license numbers
  • Passport data
  • Medical records

3. Implementing Security Controls Intermediate

Key Concepts

  • Control Objectives: Desired security state
  • Security Controls: Measures to fulfill objectives
  • Gap Analysis: Review objectives vs. actual controls

Security Control Categories

Category Description Examples
Technical Digital security enforcement Firewalls, ACLs, IPS, encryption, antivirus
Operational Process management User access reviews, log monitoring, vulnerability management, incident response, training
Managerial Risk management process Risk assessments, planning, change management, service acquisition, project management
Physical Physical world controls Fences, lighting, locks, suppression, alarms, cameras

Security Control Types

Type Purpose Examples
Preventive Stop issues before they occur Firewalls, encryption, access controls, guards
Deterrent Discourage attackers Warning signs, visible cameras, barbed wire
Detective Identify events after they occur IDS, log analysis, audits
Corrective Remediate issues after they occur Backup restoration, patching, incident response
Compensating Mitigate risk when primary controls can't be used Network isolation, extra monitoring
Directive Inform people what to do Policies, procedures, standards, guidelines, training
Control categories and types are UNIQUE to CompTIA. Memorize both the categories (Technical, Operational, Managerial, Physical) and types (Preventive, Deterrent, Detective, Corrective, Compensating, Directive). You will be tested on identifying which category and type applies to specific scenarios.

4. Data Protection Strategies Intermediate

Three Data States

State Description Vulnerabilities Protection Methods
Data at Rest Stored on media Theft, browsing, device theft Full-disk encryption, access controls, physical security
Data in Transit Moving over networks Eavesdropping, MITM, sniffing TLS/SSL, VPN, encrypted protocols, segmentation
Data in Use Processed in memory Memory dumps, cold boot, injection, screen capture App-level encryption, secure enclaves, isolation, memory encryption

Data Loss Prevention (DLP) Systems

Data Loss Prevention (DLP)

Systems that enforce information handling policies to prevent data loss and theft by searching for unsecured sensitive information and monitoring network traffic for removal attempts.

DLP Deployment Types

Type Description Capabilities
Agent-based (Host-based) Software agents on systems Search data, monitor config, block USB, control actions, auto-encrypt
Agentless (Network-based) Dedicated network devices Monitor outbound, block unencrypted, auto-encrypt, alert, email integration

DLP Detection Mechanisms

Detection Methods

  • Pattern Matching: Recognizes sensitive formats and terms
  • Watermarking: Electronic tags for tracking and DRM

5. Data Minimization and Obfuscation Techniques Advanced

Principle

Reduce risk by reducing the amount of sensitive information maintained. Destroy data when no longer needed.

Technique Description Use Cases Considerations
Hashing One-way transformation Password storage, integrity, anonymizing Vulnerable to rainbow tables, use salt, irreversible
Tokenization Replace values with random identifiers Credit cards, student IDs, medical records Lookup table must be secured, reversible, good for repeated use
Masking Partial redaction Credit card displays, SSNs, account numbers Irreversible, maintains format, good for display
Deidentification Remove ability to link data to individuals Research datasets, analytics, public reporting May allow re-identification, needs analysis, often combined

6. Access Restrictions & Network Protection Advanced

Geographic Restrictions

Limit access based on location using IP, GPS, or network boundaries.

Permission Restrictions

Limit access based on user role or authorization level. Principle of least privilege.

Network-Based Protection

Method Description Implementation
Segmentation Separate networks with controlled communication VLANs, firewalls, routing rules, DMZ
Isolation Complete cutoff from outside networks Air-gapped, no connectivity, physical separation

8. Key Definitions Summary

Must-Know Definitions

  • Gap Analysis: Review objectives vs. controls
  • Control Objectives: Desired security state
  • Security Controls: Measures to fulfill objectives
  • Data Exfiltration: Unauthorized data removal
  • Rainbow Table Attack: Precomputed hashes to reverse values
  • Compensating Controls: Alternatives when primary controls can't be used
  • Deidentification: Removing ability to link data to individuals
