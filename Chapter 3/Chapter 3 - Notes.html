1. Understanding Malware Basic

Definition: Malware

Malware is software intentionally designed to cause harm to systems, networks, or users. It can gather information, provide illicit access, and perform unwanted actions.

Common Malware Types

Type Key Characteristic Propagation Method Primary Goal
Ransomware Demands payment Phishing, RDP Financial gain
Trojan Disguised as legitimate User execution System access
Worm Self-propagating Network vulnerabilities Spreading/persistence
Virus Self-replicating User action required Spreading/payload delivery
Spyware Information gathering Various Data theft
Keylogger Input capture Various Credential theft
Rootkit Stealth/persistence Various Backdoor access

Bloatware Unwanted software Pre-installation Resource consumption

2. Ransomware Intermediate

What is Ransomware?

Ransomware encrypts files and demands payment for decryption. Crypto malware is the most common type.

Attack Vectors

  • Phishing emails with malicious attachments/links
  • RDP exploitation
  • Compromised applications

Indicators of Compromise (IoCs):

  • C&C traffic to malicious IPs
  • File encryption activity
  • Ransom notices
  • Large data transfers

Mitigation Strategies

  • Backups stored separately
  • Email security & user training
  • Endpoint protection
  • Anti-ransomware tools

3. Trojans Basic

What are Trojans?

Malware disguised as legitimate software, relying on users to run them.

Indicators of Compromise

  • Malicious application signatures
  • C&C hostnames/IPs
  • Unexpected folders/files

Mitigation Strategies

  • Security awareness training
  • Application whitelisting
  • Anti-malware/EDR tools

4. Worms Intermediate

What are Worms?

Self-propagating malware that spreads without user interaction.

Propagation Methods

  • Network vulnerabilities
  • Email attachments
  • File shares
  • USB drives

Indicators of Compromise

  • Malicious files
  • Downloads from remote systems
  • C&C contact
  • Malicious system commands

Mitigation Strategies

  • Firewalls, IPS, segmentation
  • Patch management
  • Anti-malware/EDR
  • System reimaging

5. Spyware Basic

What is Spyware?

Malware designed to obtain information about users or systems and report it to attackers.

Types of Information Targeted

  • Browsing habits
  • System configuration
  • Personal files
  • Camera/microphone access

Indicators of Compromise

  • Remote access indicators
  • Malicious processes
  • Browser injection attacks

Mitigation Strategies

  • User education
  • Software whitelisting
  • Anti-spyware tools

6. Bloatware Basic

What is Bloatware?

Unwanted applications installed by manufacturers or bundled with software.

Security Concerns

  • Vulnerabilities
  • Privacy concerns
  • Resource consumption

Mitigation Strategies

  • Uninstall unwanted apps
  • Use clean OS images
  • Deployment standards

7. Viruses Intermediate

What are Viruses?

Malicious programs that self-replicate once activated, typically requiring user action.

Virus Types

Type Characteristics Persistence
Memory-resident Remain in memory Until shutdown
Non-memory-resident Execute and shut down File-based
Boot sector Reside in boot sector Survives OS reinstall
Macro Use document macros Document-based
Email Spread via email Various

Mitigation Strategies

  • Update browsers/plugins
  • Behavioral analysis
  • IPS, reputation-based protection
  • System wipe/reinstall

8. Keyloggers Basic

What are Keyloggers?

Programs that capture keystrokes and other input for attackers.

Capture Methods

  • Kernel-level hooks
  • API interception
  • Script-based (JavaScript)
  • Memory-based

Indicators of Compromise

  • File hashes/signatures
  • Exfiltration activity
  • Disguised process names

Mitigation Strategies

  • Patching, anti-malware
  • Multi-factor authentication
  • Trusted bootable media

9. Logic Bombs Advanced

What are Logic Bombs?

Malicious code embedded in legitimate programs, triggered by specific conditions.

Characteristics

  • Embedded in applications
  • Triggered by events/dates
  • Difficult to detect

Mitigation Strategies

  • Code review
  • Secure coding practices
  • Version control/change management

10. Rootkits Advanced

What are Rootkits?

Malware designed to provide backdoor access and conceal its presence using stealth techniques.

Stealth Techniques

  • File system hooks
  • Process/network hiding
  • MBR infection

Indicators of Compromise

  • File hashes/signatures
  • C&C domains/IPs
  • Behavior-based identification

Mitigation Strategies

  • Patching, secure configs
  • External system analysis
  • System rebuild/restoration
  • Secure Boot

13. Critical Terms Summary

Must-Know Definitions

  • Ransomware: Malware demanding payment
  • Trojan: Disguised as legitimate software
  • Worm: Self-propagating malware
  • Virus: Self-replicating, needs user action
  • Spyware: Information-gathering malware
  • Keylogger: Captures input
  • Rootkit: Stealthy backdoor malware
  • Logic Bomb: Conditional activation code
  • Bloatware: Unwanted, resource-consuming software
